## chat-diff-answer-snapshots

Здесь лежат **снимки** “как на скрине”: когда в чате был промт и ответ “что сделано”, а в IDE/диффе показывалось `file +N -M`.

Зачем:
- даже если рабочая копия на сервере откатится/сломается — остаётся текстовый слепок изменений;
- можно быстро восстановить изменения, применив diff вручную.

Формат файлов:
- один файл = один логический блок изменений (например “push-timeout → wait_password redirect”).
- внутри: краткое описание контекста + сохранённый `git diff`.
